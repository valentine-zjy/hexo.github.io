3、数据链路层

一、数据链路层基础概论

1、数据链路层概念

数据链路层是计算机网络体系结构中的第二层，它负责将物理层提供的原始比特流转换成具有一定意义的数据帧，并通过物理介质将数据帧传输到相邻节点。

数据链路层的主要功能包括帧封装、透明传输、差错检测和流量控制等。它还负责管理物理层与网络层之间的数据传输，通过提供可靠的数据传输服务来保证网络通信的可靠性和高效性。常见的数据链路层协议有以太网、PPP、HDLC等。

2、帧的概念

帧是数据链路层的基本传输单位，它是数据链路层封装数据的基本格式。

帧由首部、数据和尾部三部分组成：

  1. 首部（帧头）：起始标志、源MAC地址、目的MAC地址、类型。它的作用是用于帧的定界和识别。不同数据链路层协议的帧头格式可能不同，但通常都包含了一些控制信息和地址信息，以便接收方能够正确地识别和处理帧。
  2. 数据
  3. 尾部（帧尾）：结束标志、差错检测码。它的作用是用于帧的校验和纠错。帧尾的长度通常比帧头短，它通常包含了一些控制信息和校验码，以便接收方能够检测和纠正传输过程中可能出现的错误。

之所以数据链路层能够提供提供相邻结点的可靠数据传输，正是因为帧头帧尾的有效校验和控制，确保帧的可靠传输和正确处理。

数据链路层向下还能够细分，主要分为 The logical link control (LLC) 层和 The medium access control (MAC) 层。

MAC子层负责控制物理介质的访问，而LLC子层负责提供一个可靠的数据传输服务。

MAC

MAC地址是数据帧中的一个重要部分，用于标识网络中的每个设备。它是一个48位的地址，通常表示为十六进制数字对（是独一无二的）。

作用：数据帧的封装/卸装，帧的寻址和识别，帧的接收与发送，链路的管理，帧的差错控制等。MAC子层的存在屏蔽了不同物理链路种类的差异性。

LLC

LLC是OSI模型中数据链路层的上层，位于MAC子层之上。LLC子层为网络层提供了一个可靠的数据传输服务，它负责将网络层的数据分成适当的数据块，并将这些数据块打包成帧进行传输。

作用：LLC子层的主要功能为传输可靠性保障和控制，数据包的分段与重组，数据包的顺序传输。

4、数据链路层的两种传输方式

单播和广播

5、数据链路层的三个基本问题

- 封装成帧

封装成帧就是在一段数据的前后分别添加首部和尾部，这样就构成了一个帧，接收端在收到物理层上交的比特流后，就能根据首部和尾部的标记，从收到的比特流中识别帧的开始和结束。

-透明传输

透明传输是指不管所传数据是什么样的比特组合，都应当能够在链路上传送。

-差错检测

收到正确的帧就要向发送端发送确认，发送端在一定的期限内若没有收到对方的确认，就认为出现了差错，因而就进行重传，直到收到对方的确认为止。

6、局域网中的设备

  1. 集线器：
    • 集线器是一个简单的网络设备，它的主要功能是将多个计算机连接在一起形成一个局域网。集线器的工作原理是将数据包从一个端口转发到其他所有端口，这意味着当一个计算机发送数据包时，所有连接到集线器的计算机都会收到这个数据包。因此，集线器的缺点是它不能提供任何形式的数据包过滤或分段，这可能会导致网络拥塞和安全问题。
    • 英文名称：Hub
  2. 交换机：
    • 交换机是一种更高级的网络设备，它可以在不同的网络设备之间建立虚拟连接，以便在这些设备之间传输数据。交换机可以学习每个设备的MAC地址，并根据这些地址将数据包转发到正确的端口。这使得交换机可以提供更高的网络性能和更好的安全性，因为它只会将数据包发送到目标设备，而不是广播到所有设备。
    • 英文名称：Switch
  3. 网桥：
    • 网桥是一种连接两个局域网的网络设备，它可以将两个局域网连接在一起形成一个更大的局域网。网桥的工作原理类似于交换机，但它只能连接两个网络。网桥可以学习每个设备的MAC地址，并根据这些地址将数据包转发到正确的网络。这使得网桥可以提供更高的网络性能和更好的安全性，因为它只会将数据包发送到目标网络，而不是广播到所有网络。
    • 英文名称：Bridge

二、数据链路层的通信协议

1、冲突域和广播域

冲突域

现在的设备都差不多做到了隔离冲突域，冲突域也就就逐渐消失了

交换机的每一个端口都是一个冲突域，冲突域只能发生在一个网段

分层：基于OSI的第一层物理层

广播域

交换机的所有端口都在一个广播域；广播域在一个或多个网段内发生。

分层：基于OSI的第二层数据链路层

两者的区别

1、广播域可以跨网段
2、在冲突域中所有的数据传输都是以广播形式；在广播域中，只有广播帧才会被所有主机接收。
3、冲突域是基于第一层（物理层），而广播域是机于第二层（数据链路层） 。
4、HUB 所有端口都在同一个广播域，冲突域内。Swith所有端口都在同一个广播域内，而每一个端口就是一个冲突域。

2、虚拟局域网（实验）

简介

VLAN（Virtual Local Area Network）即虚拟局域网，是将一个物理的LAN在逻辑上划分成多个广播域的通信技术。VLAN内的主机间可以直接通信，而VLAN间不能直接通信，从而将广播报文限制在一个VLAN内。

（1）虚拟局域网的优点和目的

  1. 划分广播域
  2. 增强局域网的安全性
  3. 提高健壮性
  4. 灵活构建工作组

（2）划分VLAN的方式

方法名称 具体方法
基于端口 给交换机的每个接口配置不同的PVID，当一个数据帧进入交换机接口时，如果没有带VLAN标签，且该接口上配置了PVID，那么，该数据帧就会被打上接口的PVID。如果进入的帧已经带有VLAN标签，那么交换机不会再增加VLAN标签，即使接口已经配置了PVID。
基于子网 配置好子网与VLAN映射表，如果交换设备收到的是untagged（不带VLAN标签）帧，交换设备根据报文中的源IP地址信息，确定添加的VLAN ID。将指定网段或IP地址发出的报文在指定的VLAN中传输，减轻了网络管理者的任务量，且有利于管理。
基于MAC地址 先配置好MAC地址和VLAN ID映射关系表，当终端用户的物理位置发生改变，不需要重新配置VLAN。提高了终端用户的安全性和接入的灵活性。
基于协议 将网络中提供的服务类型与VLAN相绑定，方便管理和维护。需要对网络中所有的协议类型和VLAN ID的映射关系表进行初始配置。需要分析各种协议的地址格式并进行相应的转换，消耗交换机较多的资源，速度上稍具劣势。
基于匹配策略 先在交换机上配置好终端的MAC地址和IP地址，并与VLAN关联。只有符合条件的终端才能加入指定VLAN。符合策略的终端加入指定VLAN后，严禁修改IP地址或MAC地址，否则会导致终端从指定VLAN中退出。

基于端口的方法分类：

  • Access：只允许通过一个VLAN
  • Trunk：允许通过多个VLAN
  • hybird：华为的二层接口特殊形式

（3）CSMA/CD

CSMA/CD（Carrier Sense Multiple Access with Collision Detection）是一种用于以太网的介质访问控制协议。它用于协调多个设备共享同一物理介质（如同一局域网）进行数据传输的过程。CSMA/CD确保在发送数据之前检测到冲突，并在发生冲突时采取适当的措施。

CSMA/CD协议的四大要点如下：

  1. 先听再发
  2. 边听边发
  3. 冲突停止
  4. 延迟后发

通过这些机制，CSMA/CD协议可以有效地协调多个设备在同一物理介质上进行数据传输，减少冲突的发生，并确保数据的可靠传输。然而，随着以太网技术的发展，CSMA/CD已经逐渐被更先进的协议（如以太网交换机使用的CSMA/CA）所取代，因为后者能够更好地应对网络中的冲突和碰撞问题。

（4）PPP

PPP（Point-to-Point Protocol）是一种用于在计算机网络中建立点对点连接的协议。它提供了一种可靠的、经过认证的数据链路层协议，常用于通过串行线路（如电话线或光纤）连接两个网络节点。

点对点通信是一对一信道，因此不会发生碰撞，因此比较简单，采用PPP协议；其中PPP协议就是用户计算机和ISP(互联网服务提供商)进行通信时使用的数据链路层的协议

PPP 最初设计是为两个对等节点之间的 IP 流量传输提供一种封装协议。

（5）CRC

CRC（Cyclic Redundancy Check）是一种 错误检测技术，常用于数据传输过程中对数据的完整性进行验证。CRC是一种循环冗余校验，它基于多项式除法来生成和检测校验码。

CRC的工作原理如下：

  1. 选择生成多项式
  2. 数据帧扩展
  3. 生成校验码
  4. 发送数据帧
  5. 接收端校验

CRC能够检测到多种类型的错误，包括单比特错误、双比特错误以及大部分的突发错误。然而，CRC只能检测错误，不能进行纠正。如果检测到错误，通常的做法是重新发送数据帧以确保数据的完整性。
CRC广泛应用于各种数据通信领域，如以太网、串行通信、存储介质等。它是一种快速、简单和有效的错误检测技术，能够提高数据传输的可靠性。
