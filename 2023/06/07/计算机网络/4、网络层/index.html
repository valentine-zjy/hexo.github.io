4、网络层

在计算机网络中，网络层是OSI模型中的第三层，它主要负责为数据包选择合适的路径并进行路由，以便将数据包从源主机传输到目标主机，单位：分组，提供点到点服务。

一、网络层的作用

  1. 路由：网络层通过选择合适的路径和路由协议，将数据包从源主机传输到目标主机。

  2. 寻址：网络层使用IP地址来标识网络中的主机和路由器，以便正确地将数据包传输到目标主机。

  3. 分段和重组：网络层可以将大的数据包分成较小的片段，在传输过程中重新组装这些片段，以便更好地适应网络传输的特点。

  4. 差错检测和恢复：网络层可以使用一些技术，如校验和和重传机制，来检测和纠正在传输过程中可能出现的错误。

二、网际层协议

ARP RARP ICMP IGMP

IP

IP所提供的服务通常被认为是无连接的和不可靠的，它将差错控制和流量控制之类的服务授权给了其他的各层协议，这正是TCP/IP 能够高效率工作的一个重要保证。网络层的功能主要由IP来提供，除了提供端到端的分组分发功能外，IP还提供很多扩充功能。例如，为了克服数据链路层对帧大小的限制，网络层提供了数据分块和重组功能，这使得很大的IP数据包能以较小的分组在网络上传输。

1、ARP

ARP(Address Resolution Protocol，地址解析协议)用于动态地完成IP地址向物理地址的转换。物理地址通常是指计算机的网卡地址，也称为MAC(Medai Access Control，媒体访问控制)地址，每块网卡都有唯一的地址。

2、RARP

RARP(Reverse Address Resolution Protocol，反向地址解析协议)用于动态完成物理地址向IP地址的转换。

3、ICMP

ICMP(Internet Control Message Protocol，网际控制报文协议)是一个专门用于发送差错报文的协议，由于IP协议是一种尽力传送的通信协议，即传送的数据可能丢失、重复、延迟或乱序传递，所以需要一种尽量避免差错并能在发生差错时报告的机制，这就是ICMP的功能。

4、IGMP

IGMP(Internet Group Management Protocol，网际组管理协议)允许Internet中的计算机参加多播，是计算机用做向相邻多目路由器报告多组成员的协议。多目路由器是支持组播的路由器，它向本地网络发送IGMP查询，计算机通过发送IGMP报告来应答查询。多目路由器负责将组播包转发到网络中所有组播成员。

三、IP（Internet Protocol Address）

互联网协议地址

1、IP地址的概念

  • IP地址是IP协议提供的一种统一的地址格式，它为互联网上的每一个网络和每一台主机分配一个逻辑地址，以此来屏蔽物理地址的差异。
  • IP地址的组成：一个IP地址由4个字节，32位组成，一般用点分十进制的方式表现。
  • IP地址和MAC地址的区别：
    1. 对于网络中的一些设备，路由器或者是PC及而言，IP地址的设计是出于拓扑设计出来的，只要在不重复IP地址的情况下，它是可以随意更改的；而MAC地址是根据生产厂商烧录好的，它一般不能改动的，一般来说，当一台PC机的网卡坏了之后，更换了网卡之后MAC地址就会变了。
    2. 它们最明显的区别就是长度不同，IP地址的长度为32位，而MAC地址为48位。
    3. 它们的寻址协议层不同。IP地址应用于OSI模型的网络层，而MAC地址应用在OSI模型的数据链路层。 数据链路层协议可以使数据从一个节点传递到相同链路的另一个节点上（通过MAC地址），而网络层协议使数据可以从一个网络传递到另一个网络上
    4. 分配依据不同。IP地址的分配是基于我们自身定义的网络拓扑，MAC地址的分配是基于制造商。一个是主观的，而另一种是客观的。

2、IP地址的组成

  • IP地址由网络部分和主机部分组成；

  • 主机地址/主机号：标识某一台设备的地址

  • 网络地址/网络号：标识某一个网段的地址

子网掩码：

它是一种用来指明一个IP地址的哪些位标识的是主机所在的子网，以及哪些位标识的是主机的位掩码。子网掩码不能单独存在，它必须结合IP地址一起使用。子网掩码只有一个作用，就是将某个IP地址划分成网络地址和主机地址两部分。

子网掩码的计算：

网络掩码一般与IP地址结合使用，其中值为1的比特对应IP地址中的网络位；值为0的比特对应IP地址中的主机位，以此来辅助我们识别一个IP地址中的网络位与主机位。即网络掩码中1的个数就是IP地址的网络号的位数，0的个数就是IP地址的主机号的位数。

3、IP地址的分类（常见ABC）

根据IP地址的格式和地址范围，IP地址可以分为以下五类：

  1. A类地址：以0开头，1-126为网络号，后面三个字节为主机号，可用于大型网络，最多可容纳2^24个主机。

  2. B类地址：以10开头，128-191为网络号，后面两个字节为主机号，可用于中型网络，最多可容纳2^16个主机。

  3. C类地址：以110开头，192-223为网络号，后面一个字节为主机号，可用于小型网络，最多可容纳2^8个主机。

  4. D类地址：以1110开头，用于多点广播地址，不能用于单点通信。

  5. E类地址：以1111开头，保留地址，暂时没有使用。

需要注意的是，IP地址中的前三类地址（A、B和C类）是公共地址，可以用于Internet上的任何计算机；而D类和E类地址则是特殊用途地址，不分配给具体的主机。

特殊地址：

  1. 网络地址：主机号全零地址不可用
  2. 广播地址：主机号全为一的地址
  3. 回环地址：127.0.0.0

对ABC三类地址进行区分的方式

4、IPV6

因为IPv4地址满足不了需求，出现匮乏的情况，所以诞生了IPv6地址继续使用。v6地址由128位，16个字节组成，一般表现形式为十六进制。

三、子网划分

1、为什么要子网划分？

IPV4地址如果只使用有类（A、B、C类）来划分，会造成大量的浪费或者不够用，为了解决这个问题，可以在有类网络的基础上，通过对IP地址的主机号进行再划分，把一部分划入网络号，就能划分各种类型大小的网络了。

IPV4子网划分和聚合

为了解决IPv4的不足，提高网络划分的灵活性，诞生了两种非常重要的技术，那就是VLSM（可变长子网掩码）和CIDR（无类别域间路由）
VLSM用于IPv4子网的划分，也就是把一个大的网络划分成多个小的子网；而CIDR则用于IPv4子网的聚合，当然主要是指路由方面的聚合，也就是路由汇总。通过CIDR可以把多个小的子网路由条目汇总成一个大网络的路由条目，以减少路由器中路由条目的数量，提高路由效率。

等长子网划分

等长子网划分就是将一个有类网络等分成多个网络，也就是等分成多个子网，所有子网的子网掩码都相同。

详解看上方博客，这里直接上例题：

例1：

某公司申请一个C类IP地址，这个公司有4个部门，A部门有10台主机，B部门有15台主机，C部门有30台主机，D部门有20台主机，现需要给每个部门划分一个子网。分配了一个总的网段为：192.168.2.0 , 怎么为每个部门划分单独网段。

解：

如题可知需要划分4个子网，192.168.2.0/26
可以用主机数2^6-2=62个62>30满足所需要求
IP： 192.168.2.0 11000000 10101000 00000010 00000000
子网掩码 255.255.255.192 11111111 11111111 11111111 11000000
网络地址 192.168.2.0 11000000 10101000 00000010 00000000

所以四个子网为：
子网1的网络地址：192.168.2.00000000即：192.168.2.0
子网2的网络地址：192.168.2.01000000即：192.168.2.64
子网3的网络地址：192.168.2.10000000即：192.168.2.128
子网4的网络地址：192.168.2.11000000即：192.168.2.192

例2：某公司申请到网络地址为192.3.2.0现在要划分5个部门，最大的一个部门有28台计算机，给个部门在单独子网中，则，子掩码为多少，5个部门的子网地址分别是什么。

可以划分2^3=8个子网，所以每个子网的地址分别如下：
192.3.2.000 00000即：192.3.2.0
192.3.2.001 00000即：192.3.2.32
192.3.2.010 00000即：192.3.2.64
192.3.2.011 00000即：192.3.2.96
192.3.2.100 00000即：192.3.2.128
192.3.2.101 00000即：192.3.2.168
192.3.2.110 00000即：192.3.2.192
192.3.2.111 00000即：192.3.2.224

对于上面 ”/26“ 的解释：

  "/26" 是指CIDR（Classless Inter-Domain Routing）表示法中的一个标记，用于表示子网掩码的长度或网络前缀的位数。表示：26位用于网络地址，剩余的6位用于主机地址。
  例如，假设有一个IP地址段 192.168.0.0/26，其中的子网掩码为 255.255.255.192。这意味着前26位用于网络地址，后6位用于主机地址。可用的主机地址范围是从 192.168.0.1 到 192.168.0.62，其中 192.168.0.0 是网络地址，192.168.0.63 是广播地址。

在常见的IPv4网络中，/后面的数字可以从 0 到 32，表示子网掩码中网络前缀的位数。

4、网络层的路由

- 路由

1、什么是路由？

路由（Routing）是计算机网络中的一个概念，它指的是确定数据包从源主机到目标主机所经过的路径或路线的过程，通过路由可以确认转发IP报文的路径。

路由是网络层最主要的工作任务

2、什么是路由器？

路由器（Router）是一种网络设备，用于连接多个计算机网络，并根据网络地址信息转发数据包到目标网络或目标主机。

路由器在计算机网络中起到关键的作用，它负责在不同的网络之间传递数据包，实现网络之间的通信和数据转发。路由器基于目标IP地址、路由表和路由协议等信息，根据特定的转发算法决定数据包的下一跳路径，使其能够到达目标网络或目标主机

路由器的一个端口代表一个网段，路由器中存放着通往各个网段的表格，叫做路由表。

3、路由表

路由表（Routing Table）是路由器或主机上存储的一张表格，用于存储网络地址与对应的下一跳（Next Hop）信息，以及其他相关的路由信息。路由表决定了数据包在网络中的转发路径。

路由表中的每一项记录都包含以下关键信息：

  1. 目标网络地址（Destination Network Address）：表示数据包的目标网络或目标主机的网络地址。通常使用网络前缀表示，如 IPv4 中的 CIDR 表示法（例如，192.168.1.0/24）或 IPv6 中的前缀表示法（例如，2001:db8::/32）。

  2. 子网掩码

  3. 下一跳（Next Hop）：表示数据包应该转发到的下一个设备或网络接口，以使其最终到达目标网络。下一跳可以是另一个路由器的接口地址或特定网络的出口地址。

  4. 接口（Interface）：指示数据包从哪个网络接口出去，以达到下一跳或目标网络。

  5. 路由度量值（Route Metric）：用于选择最佳路径的度量值或成本。不同的路由协议使用不同的度量值，例如跳数、带宽、延迟等。

路由表是动态更新的，可以通过静态路由配置或动态路由协议（如OSPF、BGP等）来学习和更新路由信息。

4、网关

网关（Gateway）是计算机网络中的一个重要概念，它是网络中连接两个不同网络的节点或设备。

网关在网络通信中扮演着桥梁的角色，它负责转发数据包从一个网络传输到另一个网络。网关可以是一个物理设备（如路由器）或一个逻辑实体（如软件应用程序）。

主要功能包括：

  1. 网络连接：网关允许不同的网络之间进行连接，使得数据能够在网络之间流动。

  2. 数据转发：当数据包从源网络到达网关时，网关会根据目标网络的地址信息决定将数据包转发到哪个网络。

  3. 协议转换：网关可以执行协议转换，将来自一个网络的数据包转换为适用于目标网络的协议。

  4. 数据过滤和安全性：网关可以执行数据过滤、防火墙规则和网络安全策略，确保网络流量的安全和可靠性。

  5. 网络地址转换（NAT）：网关可以执行网络地址转换，将内部私有网络的IP地址转换为外部公共网络的IP地址，实现内部网络与外部网络的通信。

  6. VPN接入：网关可以提供虚拟专用网络（VPN）接入功能，允许远程用户通过加密的连接访问私有网络资源。

以下是动态路由的特点、配置方式和示例的表格描述：

路由类型 特点 配置方式 示例

动态路由协议

以下是关于动态路由协议RIP（Routing Information Protocol）、OSPF（Open Shortest Path First）、BGP（Border Gateway Protocol）和IS-IS（Intermediate System to Intermediate System）的介绍，包括它们的特点、配置方式以及优缺点：

动态路由协议 特点 配置方式 优点 缺点
RIP - 基于跳数（跳数矩阵）的距离矢量路由协议 - 配置路由器上的网络和版本信息 - 简单、易于配置 - 路由收敛较慢，对大型网络不适用
- 最大支持跳数限制为15跳 - 配置接口上的RIP启用和版本号 - 低带宽消耗 - 容易产生路由环路 ，可能会一直传输垃圾信息
- 使用广播进行路由信息的传播 - 配置RIP网络地址和下一跳信息 - 支持VLSM（可变长子网掩码） - 路由表规模受限
- 路由更新周期默认为30秒，更新广播时使用triggered update方式
OSPF - 基于链路状态（链路状态数据库）的链路状态路由协议 - 配置区域、接口类型和相关参数 - 高可伸缩性和稳定性 - 配置复杂，对资源消耗较大
- 收集并维护链路状态数据库，使用SPF（最短路径优先）算法计算最短路径树 - 配置邻居关系和区域之间的连接 - 支持分层区域结构 - 需要较多内存和计算资源支持
- 支持分层区域结构，可适应复杂网络拓扑 - 配置区域边界路由器（ABR）和自治系统边界路由器（ASBR） - 路由更新快速 - 配置错误可能导致路由环路
- 路由更新通过洪泛方式进行 -相对于RIP限制少，适合大规模网络
BGP - 用于自治系统间的外部路由交换 - 配置自治系统号和BGP邻居关系 - 灵活的路由策略控制 - 配置复杂，需要专业知识支持
- 支持多路径、策略路由和路径属性控制 - 配置BGP路由策略和过滤规则 - 高度可扩展 - 路由表规模庞大
- 可用于连接ISP和跨多个自治系统的网络 - 配置BGP路由聚合和重分发 - 支持可靠的连接（使用TCP） - 路由收敛较慢
- 使用TCP作为可靠的传输协议进行路由信息交换
IS-IS - 基于链路状态（链路状态数据库）的链路状态路由协议 - 配置路由器ID、区域、接口和相关参数 - 高效的链路状态数据库更新 - 配置复杂，对资源消耗较大
- 使用SPF（最短路径优先）算法计算最短路径树 - 配置IS-IS邻居关系和区域之间的连接 - 支持分层网络结构 - 路由表规模受限
- 支持层2和层3的网络拓扑结构 - 配置区域边界路由器（Level 1-2 router）和自治系统边界路由器（Level 2 router） - 可适用于大型网络 - 不支持VLSM
- 路由更新通过链路状态数据库的广播方式进行

RIP和OSPF的区别：

  1. RIP是基于矢量的协议，OSPF是基于链路状态.
  2. 2.RIP适用于中小型网络拓扑，OSPF适用于较大规模的网络.
  3. 3.OSPF支持可变长度子网掩码（VLSM），RIP不支持.
  4. 4.OSPF的收敛速度比RIP的更加迅速.
  5. 5.OSPF防环，RIP不防环
